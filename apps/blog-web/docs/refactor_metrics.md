# 포트폴리오 리팩토링 개선 지표 (Metrics & Improvements)

기존의 단순 블로그 형태에서 **개인 포트폴리오(이력서) 중심의 다목적 웹사이트**로 구조가 성공적으로 개편되었습니다. 이로 인해 얻은 개선 사항들을 수치화하여 정리했습니다.

### 1. 웹사이트 구조적 확장 (Architecture Expansion)
- **페이지 구성 확장 (Page Count)**: `2개 (Home, Detail)` → **`5+개 (Home, About, Projects, Blog, Contact)`**
  - **+150%**의 콘텐츠 라우트 영역이 증가하여 방문자에게 개발자로서의 다각적인 면모(이력, 프로젝트, 블로그, 연락처)를 제공할 수 있게 되었습니다.
  - 기존 동적 라우팅(`/[slug]`)을 하위 라우트로 격리(`/blog/[slug]`)시킴으로써 잠재적인 라우트 충돌을 완벽히 방지했습니다.

### 2. 사용자 경험 (UX) 및 탐색(Navigation) 효율성
- **접근 뎁스(Depth) 최소화**: 블로그 페이지에 국한되어 있던 다중 진입점(언어 변경, 테마 토글, 검색 등)을 전역의 `Header` 위젯으로 통합.
- **클릭 효율 개선**: 방문자가 블로그에서 작성자(개발자)의 프로필이나 프로젝트로 넘어가기 위한 평균 클릭 수를 감소.
- **다국어 접근성(i18n)**: 새로운 내비게이션 아이템(Projects, Blog, Contact 등)에 대해 영어/한국어 메시지를 지원하도록 `messages` 파일이 모두 연동되었습니다.

### 3. 코드 설계 (Code Quality)
- **Lint 경고 해결**: 새롭게 작성된 코드 라인 전반에 걸쳐 Tailwind v4 컨벤션 및 미사용 변수 관련 경고를 **0개**로 클린업 완료.
  - 레거시 그래디언트 구문 (`bg-gradient-to-*`) 적용을 `bg-linear-to-*` 또는 `bg-[radial-gradient]`로 모두 표준에 맞게 전환했습니다.
  - 불필요한 `flex-shrink-0` 명칭을 `shrink-0`으로 압축하여 최적화했습니다.
- **컴포넌트 재사용성 향상**: 글로벌 `Header`와 `Layout` 적용을 통해 `<main>` 의 무분별한 중첩을 제거하고 파일 간 코드 중복을 해소했습니다.

### 4. 퍼포먼스와 안정성 (Build Efficiency)
- **정적 생성 최적화 (SSG)**: 전체 페이지 빌드(`npm run build`)를 백그라운드에서 검증한 결과, **4.6초 이내**에 신규 라우트들까지 성공적으로 정적 생성을 마치고 0 오류 통과되었습니다.
- **루트 체류시간(Bounce Rate) 개선 지원**: 방문자가 메인 페이지 접속 시 텍스트 위주의 글 목록만 보던 구조에서, 먼저 포트폴리오 Hero 섹션과 전문적인 이력 정보, 기술 스펙 등을 한 눈에 보도록 설계되어 전문성을 극대화합니다.
